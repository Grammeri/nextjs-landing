# Design Tokens System

Этот документ описывает системный стандарт использования дизайн-токенов и относится к внутренней разработческой документации, определяющей архитектурные инварианты. Документ читается в IDE.

Документ не используется для публичной документации, маркетинговых материалов или onboarding-гайдов.

---

## Purpose

Система обеспечивает визуальную согласованность, предсказуемое поведение UI, масштабируемость и безопасные изменения.

## Architecture

Design tokens описывают цвета, поверхности, тени, motion и hover-эффекты, а также интерактивное поведение.

Система токенов разделена на несколько логических слоёв.

**Primitive tokens** — базовые атомарные значения, не используемые напрямую в компонентах.

Примеры:
```css
--color-red-500
--opacity-90
--scale-sm
```

**Semantic tokens** описывают назначение значения, а не его физическую форму, и используются в компонентах.

Примеры:
- `--color-primary`
- `--surface-card`
- `--text-secondary`

**Interaction tokens** описывают интерактивное поведение и используются для hover, focus, active и motion.

Примеры:
- `--interaction-hover-opacity`
- `--interaction-hover-scale`
- `--interaction-hover-translate-y`

**Alias tokens** связывают semantic и interaction токены с primitive значениями и позволяют менять поведение системы без изменений компонентов.

## Workflow

### Добавление нового токена

Чеклист добавления нового токена:

- Определи тип токена: primitive, semantic или interaction.
- Проверь, существует ли похожий токен.
- Уточни, нужен ли новый слой.
- Добавь токен в `globals.css` с понятным неймингом.
- Используй токен в компонентах вместо raw значений.

## Edge cases

Интерактивные токены нужны для исключения магических чисел в hover-стилях.

Плохой пример:
```css
.button:hover {
  transform: scale(1.03);
  opacity: 0.9;
}
```

Проблемы:
- значение не переиспользуемо
- сложно изменить глобально
- нет системного смысла

Хороший пример:
```css
.button:hover {
  transform: scale(var(--interaction-hover-scale));
  opacity: var(--interaction-hover-opacity);
}
```

Преимущества:
- единое поведение
- централизованное управление
- безопасные изменения

## Rules and constraints

Запрещено: использовать raw значения в UI, добавлять случайные hover-числа и дублировать поведение между компонентами.

Разрешено: использовать только semantic и interaction токены, добавлять новые токены через систему и переопределять поведение через alias.

## Verification

Система токенов является частью архитектуры проекта.

Любые изменения должны не ломать существующий UI, быть обратимо управляемыми и соответствовать принципам системы.

## Summary

Design tokens — фундамент визуальной архитектуры проекта.
